## 0. このサイトの存在意義（最重要・最初に読め）

このサイトは単なる会社案内ではない。**「サイトの出来そのものが、Grayscaleの技術力の証明」**になるメタ構造を持つ。

訪問者にこう感じさせることがゴール：

> 「このサイトをこのクオリティで自分たちで作れる会社が選んだSaaSなら、間違いない。導入も任せられる。」

つまり**サイトのクオリティ＝営業資料**。中途半端な出来は、それ自体がブランド毀損になる。全国の先進的なデザインスタジオ（Awwwards受賞級）に並ぶ格を、本気で目指す。

## 1. 会社の本質（ヘッドラインの設計はここから導く）

### ビジネスの構造
```
売上のメイン     : SmartHR・Google Workspace 等のSaaS代理店業務
他社代理店との差 : 「自分が現場で製造業DXを10年やってきた」から、
                  紹介で終わらず "導入から定着まで伴走するコンサル・実装" ができる
信念             : 「本当にいいものしか紹介しない」という自負
唯一無二の証明   : 自社でシステムも作れる技術力 → そのHPで証明する
```

### サイトの主役（ヘッドラインの主語）
**「代理店 × コンサル × 開発を全部やる唯一無二の企業」というポジショニングそのもの。**

- ただのSaaS再販業者ではない。「選定 → 導入 → 定着 → 内製化支援 → 必要なら自社開発」まで一気通貫でやれる稀有な存在であることを、最も大きく押し出す。
- 「作れる会社がSaaSを選ぶから本物」というロジックを主軸に据える。

### 地域性（茨城）の扱い
**完全に外す。** 茨城・水戸・地域限定の文言や県シルエット画像（`ibaraki.png`）は新サイトでは一切使わない。全国区の先進スタジオとして勝負する。
※会社概要・所在地表記としての住所記載のみ可（フッターや会社概要モーダル）。主張やヒーローには出さない。

## 3. デザイン方針（ファクトチェック済みトレンドに基づく）

### 大原則：「白黒であること」自体は武器にならない
リサーチで「モノトーン受賞サイトは色ではなく技術/モーションで差別化している」という通説は**明確に反証された（投票 0-3）**。
社名がGrayscaleなのでモノトーン基調で行くのは正しいが、**色を捨てた分を以下3つで埋めなければ"地味なだけ"のサイトになる**：

#### 柱A — タイポグラフィで殴る（最優先投資先）
- 見出しは**巨大な可変サンセリフ（variable font）**。極太(800-900)×極細(100-300)のコントラストで緊張感を作る。
  - 候補: `Inter`, `Geist`, `Satoshi`, `General Sans` など（欧文）
- **旧サイトの `font-mono`（JetBrains Mono）を見出しに使うのをやめる。** これが「エンジニアの個人サイト」感の主因。モノは使うならラベル・キャプション等の脇役に限定。
- 日本語は `Noto Sans JP` 継続可。ただし **letter-spacing を詰めて**緊張感を出す。
- 2025-2026のトレンド "Typographic Statements"（巨大文字だけで物語る）を体現する。画像に頼らず文字で成立させる。

#### 柱B — モーションを一本の慣性で貫く
- **ページ全体にスムーススクロールを敷く**（Lenis が事実上の標準。4kb未満、GTA VI / Microsoft Design / Shopify Supply 等で採用）。
- **スクロール連動ストーリーテリング**で、「現場でDXをやってきた → だから本物の選定ができる → 導入も実装もできる」という物語を縦スクロールで展開する。
- 旧サイトの問題＝モーションが部品ごと（カードは3Dチルト、テキストはreveal…）でバラバラ。**ページ全体を貫く一貫した慣性**を最優先で持たせる。
- ⚠️ **scrolljacking（スクロール速度・方向の乗っ取り）は厳禁。** NN/g調査で大半のユーザーが混乱・方向感覚を失うと報告。慣性は効かせるが、ユーザーのスクロール制御は奪わない。`prefers-reduced-motion` を尊重すること。

#### 柱C — 信頼を「数字・ロゴ・事例」で見せる
- 「製造業で10年DX」を**具体的な定量実績**に翻訳する（例: 作業時間◯%削減、◯社導入、◯時間/月の削減 など）。
  - ※実数値はクライアント（星さん）に確認が必要。守秘で社名が出せなければ「製造業A社・従業員50名」等の匿名事例で十分効く。
- 取り扱いSaaSのロゴ（SmartHR / Google Workspace 等）を**正規の利用許諾・ブランドガイドラインの範囲で**掲示。代理店としての信頼の可視化。
- ベンチマーク: **ベイジ（baigie.me）** — B2B受託で「信頼×先進性」を両立する教科書。顧客満足度4.53・流入10倍等の定量エビデンスの見せ方を参考にする。

### トンマナの方向
- モノトーン基調（白／黒／グレースケール）。ただし**差し色を1色だけ**入れる選択肢は検討してよい（記憶に残るブランドカラー）。最終判断は実装者がモックで比較して決める。
- 余白を大胆に取る。情報を詰め込まない。「自信あるミニマル」。
- フルリニューアル＝色・フォント・レイアウト・モーションを全部ゼロベースで見直す。既存コンポーネントの流用は「結果的に意図に合えば」程度で、縛られない。

## 4. コンテンツ構成（推奨セクション順）

旧サイトの「サービス9個カルーセル」は**廃止**。全部見せて1個も刺さっていない。3つに絞って深く語る。

1. **ヒーロー** — 巨大タイポで唯一無二のポジショニングを宣言。「代理店 × コンサル × 開発」を一文で。
2. **なぜGrayscaleは"本物"なのか** — 「現場で10年DXをやった人間が選ぶSaaS」という信頼の根拠をストーリーで。
3. **3つの提供価値**（例: ①最適なSaaS選定・代理店 ②導入〜定着の伴走コンサル ③必要なら自社開発）— カルーセルでなく、それぞれを大きく深く。
4. **実績・数字**（柱C）— 定量エビデンス＋取り扱いSaaSロゴ。
5. **このサイト自体が証明**（任意・効けば強い）— 「このサイトも自分たちで作りました」とメタに言及するセクションがあると刺さる。
6. **お問い合わせ** — 既存の ContactForm / SafeQuickContact の仕組みは流用検討可。
7. **フッター／会社概要** — 確定情報（§2）を正確に。

## 5. 技術スタック

### 維持
- Next.js 14 / TypeScript / Tailwind CSS / Framer Motion

### 追加（推奨）
- **Lenis**（スムーススクロール）— `npm install lenis`。旧 `@studio-freight/*` は非推奨なので使わない。
- **GSAP + ScrollTrigger**（スクロール連動演出）— `npm install gsap @gsap/react`。
- ⚠️ Lenis と GSAP ScrollTrigger は既定で別RAFループのため放置するとjitterが出る。**Lenisのrafをgsap.tickerに統合し、マウント後に `ScrollTrigger.refresh()` する**こと（公式READMEのパターン）。

### 入れない（明確な非推奨）
- **Three.js / WebGL / 3D演出は入れない。** 理由: 小規模スタジオ×B2B受託という文脈では保守コスト・パフォーマンス負荷が高く、信頼感を優先すべき場面で過剰。タイポとモーションだけでAwwwards級は十分狙える。どうしても入れたくなったら、Lighthouseスコアとのトレードオフを先に検証してから判断する。

### 品質ガードレール
- **Core Web Vitals / Lighthouse を犠牲にしない。** 没入演出とパフォーマンスは両立させる。B2B受注はSEOにも効くため。
- レスポンシブ必須（モバイルでヒーローのタイポが破綻しないこと）。
- `prefers-reduced-motion` 対応必須。

## 8. やってはいけないこと（NGリスト）

- ❌ 茨城・地域限定の主張をヒーローや本文に出す

- ❌ scrolljackingでスクロールを乗っ取る
- ❌ Three.js/3Dを「かっこいいから」で入れる
- ❌ パフォーマンス（Lighthouse）を演出のために犠牲にする
- ❌ クオリティの低い中途半端な仕上がりで公開する（このサイトは技術力の証明そのもの）

## 9. 成功の判定基準

1. 初見で「この会社、ただの代理店じゃないな。技術がある」と伝わる。
2. 「現場でDXをやってきたから選定が本物」という信頼のロジックが、スクロール体験で腹落ちする。
3. サイトの完成度が、全国の先進スタジオと並べても見劣りしない（Awwwards級の格）。
4. モバイルでも破綻せず、Lighthouseスコアが健全。
5. 「このサイトを作れる会社が選んだSaaSなら任せたい」という問い合わせ動機を生む。
